SQL简介

SQL在软件开发方面有诸多应用,学习一个东西之前我们得先知道我们学习的目的，所以我们学习SQL的目的是什么呢？做一名Developer还是DBA? 还是作为数据分析的工具？anyway,明确自己的学习目的就好，不过这里主要以用SQL在数据分析领域中的应用为目的。数据分析？，那么数据分析的目的又是什么呢，或者说分析后能给我带来什么呢？
现今，很多公司企业都拥有着大量的数据(数据即财富)，他们希望各部门的职员都能有条不紊地利用这些数据 进行决策，如何做到呢，绝大多数数据分析师会告诉你大部分工作都是在SQL中完成的，so let‘s get start!

为什么使用SQL

为什么企业喜欢使用数据库

可以快速访问数据-SQL 可使我们从数据库中快速获取结果。 可以优化代码，快速获取结果

可以很容易共享数据 - 多个人可以访问存储在数据库中的数据，所有访问数据库的用户获得的数据都是一样。

数据库如何存储数据

如果你用过Excel你应该已经很熟悉表了 他们和数据表很像，表具有行和列 就像Excel,但是有些更严格的规范，比如数据表每列必须有独一无二的列名 如下图

看上去数据库并不知道每个列的列名指代的是什么，所以我们需要尽可能的让我们的列名更具描述性，如name表示名字，age表示年龄等等

总结下来，SQL数据库中存储数据有以下特点:

  • 数据库中的数据存储在类似于 Excel 电子表格的表中。
    大多数情况下，可以将数据库视为一堆 Excel 电子表格。每个电子表格都有行和列。每行保存有关交易、个人、公司等的数据。而每列所保存的数据与你关心的某一特定行相关，如名称、位置、唯一身份等。

  • 同一列中的所有数据必须符合数据类型。
    如果其中一列存储的是数字，如年龄，那么这一列就应该严格遵守数字这种数据类型，不能是文本或是其他类型的数据

  • 列类型一致是快速使用数据库的主要原因之一。
    数据库通常会存储海量数据。因此，知道这些列都是相同类型的数据意味着可快速从数据库获取数据。

SQL语句

说了半天，简单介绍了SQL是什么，那么接下来我们需要知道如何使用SQL,简单来说，SQL就是一门服务我们获取数据的工具，使用这个工具并不复杂，就像学会开车并不复杂，复杂的是车的构造(所以SQL的构造原理也非常复杂，还好我们不用关心)，我们只需要照着”说明书”来使用SQL这个强大的工具吧。
我们来看”说明书”的第一点：SQL语句

可以把语句理解为撰写正确的SQL代码，千万别被代码这两个字吓跑了，其实很简单，有多简单，就跟你说话一样简单(如果你有点英文基础就更好理解了) 语句会告诉数据库你想对数据库进行什么操作 以CREATE
开头的语句为例，该语句能让你在数据库中创建表;DROP TABLE语句则能让你从数据库中删除表，本文重点介绍查询数据的语句 SELECT ,为什么是SELECT,很简单，因为使用频繁，频繁，最频繁，闲着没事你肯定不会去删除表(如果你删了，那么恭喜你也被公司删了)^_^

LIMIT

我们已经学习了 SELECT（选择列）和 FROM（选择表）语句。如果想要查看表的前几行时，LIMIT 语句就能派上用场。这可比加载整个数据集要快得多。

许多时候，查询的目的只是要看看某个表是什么样的，大概有什么字段(数据列),想对字段进行什么操作，比如你想看看google的网站流量，第一步你需要弄清网站流量表里存储了什么数据，要做到这一点你只需要看几行数据就可以了，使用LIMIT子句就能轻易地缩短查询结果的返回时间，毕竟当数据量非常庞大时，一次性就像全部查询出来也是很耗费时间的，让我们用LIMIT来看网站流量表的前十行,把LIMIT写在查询的最后，后面加上数字，这样就限制了返回的行数，比如LIMIT 10就会使查询只返回前十行结果，省略其它
